How to Train an Image Classifier with RegNet

A simple step-by-step cheatsheet for complete beginners: prepare images, load a pretrained RegNet, replace the last layer, train, validate, and save your model.

What is RegNet?

RegNet is a family of convolutional neural networks designed to be simple, efficient, and strong for image classification tasks.

1) What you need first

Python PyTorch torchvision Images in folders GPU optional
  • Install: torch and torchvision
  • Know your class names, like cat and dog
  • Use a GPU if available, but CPU also works for learning

2) Big picture flow

Prepare dataset
Split train / val
Load pretrained RegNet
Replace classifier head
Train → Validate → Save

Tip: Start with a small experiment first. Make sure the whole pipeline runs before training for many epochs.

3) Step-by-step guide

1

Prepare your dataset

Put images into folders by class name.

data/ cats/ cat1.jpg cat2.jpg dogs/ dog1.jpg dog2.jpg
2

Make a train/validation split

Use most images for training and some for checking performance.

  • Common split: 80% train / 20% val
  • Do not mix the same image into both sets
3

Add transforms

Resize images and convert them to tensors. For training, light augmentation can help.

  • Resize
  • Random flip (train only)
  • Normalize
4

Load a pretrained RegNet

Starting from pretrained weights is easier and faster than training from scratch.

from torchvision.models import regnet_y_400mf model = regnet_y_400mf(weights="DEFAULT")
5

Replace the final classifier head

The pretrained model predicts ImageNet classes. You must swap the last layer to match your own classes.

import torch.nn as nn num_classes = 2 in_features = model.fc.in_features model.fc = nn.Linear(in_features, num_classes)
6

Choose loss + optimizer

For normal classification, a simple default is:

criterion = nn.CrossEntropyLoss() optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)
7

Train in a loop

For each batch: forward pass → loss → backward pass → optimizer step.

  • model.train()
  • optimizer.zero_grad()
  • loss.backward()
  • optimizer.step()
8

Validate after each epoch

Check how well the model performs on unseen validation images.

  • Use model.eval()
  • Turn off gradients with torch.no_grad()
  • Track loss and accuracy
9

Save your model

Save the learned weights so you can reuse them later.

torch.save(model.state_dict(), "regnet_classifier.pth")

4) Tiny PyTorch example

import torch import torch.nn as nn from torch.utils.data import DataLoader, random_split from torchvision import datasets, transforms from torchvision.models import regnet_y_400mf # Device device = "cuda" if torch.cuda.is_available() else "cpu" # Transforms transform = transforms.Compose([ transforms.Resize((224, 224)), transforms.ToTensor(), ]) # Dataset full_dataset = datasets.ImageFolder("data", transform=transform) num_classes = len(full_dataset.classes) # Split train_size = int(0.8 * len(full_dataset)) val_size = len(full_dataset) - train_size train_ds, val_ds = random_split(full_dataset, [train_size, val_size]) train_loader = DataLoader(train_ds, batch_size=32, shuffle=True) val_loader = DataLoader(val_ds, batch_size=32) # Model model = regnet_y_400mf(weights="DEFAULT") in_features = model.fc.in_features model.fc = nn.Linear(in_features, num_classes) model = model.to(device) criterion = nn.CrossEntropyLoss() optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)

5) Training loop basics

for epoch in range(5): model.train() for images, labels in train_loader: images, labels = images.to(device), labels.to(device) optimizer.zero_grad() outputs = model(images) loss = criterion(outputs, labels) loss.backward() optimizer.step() model.eval() correct, total = 0, 0 with torch.no_grad(): for images, labels in val_loader: images, labels = images.to(device), labels.to(device) outputs = model(images) preds = outputs.argmax(dim=1) correct += (preds == labels).sum().item() total += labels.size(0) print("Val acc:", correct / total)

6) Validation and evaluation

  • Training accuracy tells you how well the model fits seen data.
  • Validation accuracy tells you how well it generalizes to new data.
  • If training accuracy goes up but validation stays poor, you may be overfitting.
  • Look at a few wrong predictions manually — this teaches you a lot.
Good sign:
train and validation both improve
Warning:
validation stops improving early
Fixes:
more data, augmentation, lower learning rate

7) Save and reuse later

# Save weights torch.save(model.state_dict(), "regnet_classifier.pth") # Later: rebuild model, replace head, then load model = regnet_y_400mf(weights=None) model.fc = nn.Linear(model.fc.in_features, num_classes) model.load_state_dict(torch.load("regnet_classifier.pth")) model.eval()

Also save your class names, so you know which output index matches which label.

8) Common beginner mistakes

Wrong folder structure: ImageFolder expects one folder per class.
Final layer not replaced: model output size must match your number of classes.
Train/val leakage: the same or nearly identical images appear in both splits.
No model.eval() during validation: this can make results unstable.
Forgetting normalization/transforms: bad input often means bad training.
Learning rate too high: loss jumps around or does not improve.
Tiny dataset expectations: very small datasets may not generalize well.
Only checking accuracy: also inspect examples and class balance.
